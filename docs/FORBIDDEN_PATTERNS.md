# FORBIDDEN PATTERNS

## Назначение

Этот документ перечисляет запрещённые паттерны и типовые ошибки, которые особенно вероятны на маленькой локальной модели.

## Data Access Patterns

### Запрещено

- Использовать JsonPath вместо прямого доступа к `wf.*`.
- Изобретать другие корни данных, когда в задаче явно нужны `wf.vars.*` или `wf.initVariables.*`.
- Смешивать `wf.vars.*` и `wf.initVariables.*` в одном ответе без оснований из входа.

### Вместо этого

- Использовать прямой путь данных.
- Сохранять исходный root, подтверждённый входным контекстом.

## Output Violations

### Запрещено

- Добавлять prose вне допустимого output mode.
- Возвращать markdown fences вокруг Lua.
- Смешивать raw Lua и JSON в одном ответе без явного mode-switch.
- Ломать wrapper `lua{...}lua` или экранирование JSON-строки.

### Вместо этого

- В `raw_lua` возвращать только Lua.
- В `json_wrapper` и `patch_mode` возвращать валидный JSON-объект.

## Patch-Mode Violations

### Запрещено

- Переписывать весь payload вместо локального additive update.
- Возвращать неструктурированный diff-текст.
- Добавлять пустые placeholder-поля, которые не несут доменной нагрузки.

### Вместо этого

- Возвращать только нужные поля для добавления или обновления.
- Сохранять объектную форму результата.

## Array And Collection Errors

### Запрещено

- Терять array semantics после фильтрации или нормализации.
- Возвращать объект вместо массива, когда downstream ожидает массив.
- Игнорировать `_utils.array.new()` при сборке нового массива.

### Вместо этого

- Создавать новый массив через `_utils.array.new()`, если результат собирается заново.
- Явно нормализовать одиночные элементы до массива, если этого требует задача.

## Nil / Empty Handling Errors

### Запрещено

- Приравнивать `nil`, `""` и "данные есть" без явной проверки.
- Падать на пустых значениях там, где нужен безопасный fallback.

### Вместо этого

- Проверять `nil` и пустые строки отдельно.
- Явно описывать fallback-логику, если она нужна по задаче.

## Date/Time Errors

### Запрещено

- Делать разбор даты/времени без валидации формата.
- Игнорировать границы строк и timezone offset.
- Молчаливо возвращать неверный формат вместо fail-fast или safe fallback.

### Вместо этого

- Проверять формат входа перед конвертацией.
- Явно учитывать `offset_sign`, `offset_hour`, `offset_min`, если задача опирается на ISO 8601.

## Typical Small-Model Failure Modes

- выбрать неправильный root path;
- перепутать archetype и output mode;
- вернуть внешне правдоподобный, но невалидный `lua{...}lua`;
- использовать слишком общий Lua вместо LocalScript-специфичного доступа;
- проигнорировать patch-ограничение "не переписывать всё";
- пропустить edge cases с пустыми значениями и массивами.
